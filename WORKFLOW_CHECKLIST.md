# Villa Images Upload - Workflow Checklist

Use this checklist to guide you through the complete villa image upload process.

## Phase 1: Preparation

- [ ] Locate Google Drive folder with villa images (Ashoka: 20 images, Bougainville: 21 images)
- [ ] Verify all 41 images are accessible and properly named
- [ ] Note which image would be best as hero for each villa
- [ ] Open terminal/command prompt
- [ ] Navigate to project directory: `cd /c/Users/Adit/Downloads/bhumi-lovina-website`
- [ ] Verify Python 3.6+ is installed: `python --version`
- [ ] Verify Pillow or ImageMagick is available (script will auto-use either)

## Phase 2: Collect File IDs

- [ ] For each image in Google Drive, copy the file ID from the URL:
  - Go to file
  - Copy URL from address bar
  - Extract ID: `https://drive.google.com/file/d/[THIS_IS_ID]/view`

**Shortcut for bulk extraction:**
- [ ] If images are in a shared folder, use Google Drive API to export metadata (advanced)
- [ ] Otherwise, manually collect IDs for all 41 images

**Result needed:** List of 41 file IDs (20 for Ashoka, 21 for Bougainville)

### Alternative: Use Helper Script
If you have file IDs in a text file:
```bash
python scripts/extract-drive-ids.py your_file_ids.txt
```
- [ ] Script creates `villa-config.py` with formatted configuration
- [ ] Review generated config for accuracy
- [ ] Verify `hero_index` values (which image is best for each villa)

## Phase 3: Configure Processing Script

**Option A: Manual Configuration**
- [ ] Open `scripts/process_images.py` in text editor
- [ ] Locate `VILLAS` dictionary
- [ ] For Ashoka:
  - [ ] Add all 20 file IDs and image names
  - [ ] Set `hero_index` to best image (0-19)
- [ ] For Bougainville:
  - [ ] Add all 21 file IDs and image names
  - [ ] Set `hero_index` to best image (0-20)
- [ ] Save file

**Option B: Copy from Generated Config**
- [ ] Open `villa-config.py` (generated by helper script)
- [ ] Copy the `VILLAS` dictionary
- [ ] Paste into `scripts/process_images.py`
- [ ] Verify format is correct

**Verification Checklist:**
- [ ] 20 files listed for Ashoka
- [ ] 21 files listed for Bougainville
- [ ] All file IDs are 25+ characters (Google Drive format)
- [ ] All image names are set (e.g., 'IMG_7838')
- [ ] `hero_index` values are reasonable (0-20 for Ashoka, 0-20 for Bougainville)

## Rollback Plan (If Needed)

If something goes wrong:

1. **Images not uploading:**
   - [ ] Verify token is valid
   - [ ] Check file IDs are correct
   - [ ] Ensure internet connection is stable
   - [ ] Retry processing script

2. **URLs not working:**
   - [ ] Wait 5 minutes for CDN propagation
   - [ ] Clear browser cache
   - [ ] Try different browser
   - [ ] Check blob.vercel-storage.com status

3. **Components broken:**
   - [ ] Revert to old `src/lib/images.ts` (git checkout)
   - [ ] Remove references to `VILLA_IMAGES`
   - [ ] Keep generated URLs for future use

4. **Performance regressed:**
   - [ ] Check image sizes are reasonable
   - [ ] Verify correct responsive sizes being served
   - [ ] Check for cache-busting issues
